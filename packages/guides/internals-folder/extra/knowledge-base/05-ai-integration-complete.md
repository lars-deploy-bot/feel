## Lovable AI Gateway

### What is Lovable AI?

Lovable AI is a gateway service that provides access to external AI models for user applications. It's separate from the AI that powers the Lovable editor itself.

**Key Features:**
<!-- SUPABASE DISABLED: - Pre-configured API key in Supabase environments -->
- No setup required for users
- Access to Google Gemini and OpenAI GPT-5 models
<!-- SUPABASE DISABLED: - Works with Supabase and Lovable Cloud -->

**IMPORTANT:** Don't create mock AI apps - use Lovable AI immediately unless user specifically requests a mock.

---

### Available Models

**Google Gemini Models:**

1. **google/gemini-2.5-pro**
   - Top-tier in Gemini family
   - Strongest at image-text + big context + complex reasoning
   - Best for: Visual + text, heavy reasoning, large input sizes

2. **google/gemini-2.5-flash** ⭐ **DEFAULT**
   - Balanced choice: less cost & latency than Pro
   - Good on multimodal + reasoning
   - Slightly less precision in hardest cases

3. **google/gemini-2.5-flash-lite**
   - Fastest + cheapest of Gemini 2.5 line
   - Good for: Classification, summarization, simple workloads
   - Weakest on nuance and complex reasoning

4. **google/gemini-2.5-flash-image-preview** (Nano banana)
   - Creates images based on text prompts

**OpenAI GPT-5 Models:**

1. **openai/gpt-5**
   - Powerful all-rounder
   - Excellent reasoning, long context, multimodal
   - Best when accuracy & nuance matter
   - Most expensive and slower

2. **openai/gpt-5-mini**
   - Middle ground: lower cost & latency
   - Keeps most reasoning & multimodal strengths
   - Good where you need strong performance without overpaying

3. **openai/gpt-5-nano**
   - Designed for speed & cost saving
   - Lower performance on hard reasoning
   - Very efficient for high-volume/simple tasks

### Model Selection Rules

**Default Model:** `google/gemini-2.5-flash` (unless user asks for specific model)

**Model Equivalents:**
- `openai/gpt-5` → `google/gemini-2.5-pro`
- `openai/gpt-5-mini` → `google/gemini-2.5-flash`
- `openai/gpt-5-nano` → `google/gemini-2.5-flash-lite`

**CRITICAL:**
- Do NOT use Lovable AI if project already has API keys for existing LLM providers and logic implemented (unless user specifically asks)
- **Use Lovable AI by default** if there's no existing LLM implementation

### Implementation Requirements

**CRITICAL Rules:**

1. **No direct calls:** Always use the Lovable AI Gateway through edge function - never call AI models directly
2. **Don't specify prompts on client side:** If you need to update the prompt, tweak it on the backend
3. **Multiple use cases:** Create an edge function for each use case OR add branching logic inside existing edge function

**API Key Handling:**

- **NEVER ask users to provide or configure `LOVABLE_API_KEY`**

### Rate Limits

**Rate limit triggers:**
- Too many requests in short period → `429 Too Many Requests`
- Out of credits → `402 Payment Required`

**CRITICAL:** YOU MUST SURFACE THESE ERRORS
- Catch them in edge functions
- Pass them back to clients
- Display toasts in end user app when they happen

**Solution:** Add time between requests to avoid rate limiting

**Increasing rate limits:**
- Free plan → Upgrade to paid plan for higher limits
- Still not enough → Contact support@lovable.dev

**Learn more:** https://docs.lovable.dev/features/ai

## Streaming Chat Implementation

### Core Rules

1. Always call Lovable AI via backend function (never from client)
2. Keep secrets on backend (`LOVABLE_API_KEY` is auto-provisioned)
3. Stream with SSE and render tokens as they arrive
4. Parse SSE line-by-line, handle partial JSON safely
5. Flush leftovers at the end

---

### Backend Edge Function

<!-- SUPABASE DISABLED: **File:** `supabase/functions/chat/index.ts` -->

```typescript
import { serve } from "https://deno.land/std@0.168.0/http/server.ts";

const corsHeaders = {
  "Access-Control-Allow-Origin": "*",
  "Access-Control-Allow-Headers": "authorization, x-client-info, apikey, content-type",
};

serve(async (req) => {
  if (req.method === "OPTIONS") return new Response(null, { headers: corsHeaders });

  try {
    const { messages } = await req.json();
    const LOVABLE_API_KEY = Deno.env.get("LOVABLE_API_KEY");
    if (!LOVABLE_API_KEY) throw new Error("LOVABLE_API_KEY is not configured");

    const response = await fetch("https://ai.gateway.lovable.dev/v1/chat/completions", {
      method: "POST",
      headers: {
        Authorization: `Bearer ${LOVABLE_API_KEY}`,
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
        // Optional: model: "google/gemini-2.5-flash",
        messages: [
          { role: "system", content: "You are a helpful AI assistant. Keep answers clear and concise." },
          ...messages,
        ],
        stream: true,
      }),
    });

    if (!response.ok) {
      if (response.status === 429) {
        return new Response(JSON.stringify({ error: "Rate limits exceeded, please try again later." }), {
          status: 429,
          headers: { ...corsHeaders, "Content-Type": "application/json" },
        });
      }
      if (response.status === 402) {
        return new Response(JSON.stringify({ error: "Payment required, please add funds to your Lovable AI workspace." }), {
          status: 402,
          headers: { ...corsHeaders, "Content-Type": "application/json" },
        });
      }
      const t = await response.text();
      console.error("AI gateway error:", response.status, t);
      return new Response(JSON.stringify({ error: "AI gateway error" }), {
        status: 500,
        headers: { ...corsHeaders, "Content-Type": "application/json" },
      });
    }

    return new Response(response.body, {
      headers: { ...corsHeaders, "Content-Type": "text/event-stream" },
    });
  } catch (e) {
    console.error("chat error:", e);
    return new Response(JSON.stringify({ error: e instanceof Error ? e.message : "Unknown error" }), {
      status: 500,
      headers: { ...corsHeaders, "Content-Type": "application/json" },
    });
  }
});
```

### Frontend - Token-by-Token Streaming

```typescript
// Never hardcode URL - use environment variables
<!-- SUPABASE DISABLED: const CHAT_URL = `${import.meta.env.VITE_SUPABASE_URL}/functions/v1/chat`; -->

type Msg = { role: "user" | "assistant"; content: string };

async function streamChat({
  messages,
  onDelta,
  onDone,
}: {
  messages: Msg[];
  onDelta: (deltaText: string) => void; // called for each token chunk
  onDone: () => void;
}) {
  const resp = await fetch(CHAT_URL, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
<!-- SUPABASE DISABLED:       // Optional (use VITE_SUPABASE_PUBLISHABLE_KEY, NOT deprecated VITE_SUPABASE_ANON_KEY): -->
<!-- SUPABASE DISABLED:       Authorization: `Bearer ${import.meta.env.VITE_SUPABASE_PUBLISHABLE_KEY}`, -->
    },
    body: JSON.stringify({ messages }),
  });
  
  // Handle 402 and 429 errors - inform users
  if (!resp.ok || !resp.body) throw new Error("Failed to start stream");

  const reader = resp.body.getReader();
  const decoder = new TextDecoder();
  let textBuffer = "";
  let streamDone = false;

  while (!streamDone) {
    const { done, value } = await reader.read();
    if (done) break;
    textBuffer += decoder.decode(value, { stream: true });

    // Process line-by-line as data arrives
    let newlineIndex: number;
    while ((newlineIndex = textBuffer.indexOf("\n")) !== -1) {
      let line = textBuffer.slice(0, newlineIndex);
      textBuffer = textBuffer.slice(newlineIndex + 1);

      if (line.endsWith("\r")) line = line.slice(0, -1);   // handle CRLF
      if (line.startsWith(":") || line.trim() === "") continue; // SSE comments/keepalive
      if (!line.startsWith("data: ")) continue;

      const jsonStr = line.slice(6).trim();
      if (jsonStr === "[DONE]") {
        streamDone = true;
        break;
      }

      try {
        const parsed = JSON.parse(jsonStr);
        const content = parsed.choices?.[0]?.delta?.content as string | undefined;
        if (content) onDelta(content); // emit token(s) immediately
      } catch {
        // Incomplete JSON split across chunks: put it back and wait for more data
        textBuffer = line + "\n" + textBuffer;
        break;
      }
    }
  }

  // Final flush
  if (textBuffer.trim()) {
    for (let raw of textBuffer.split("\n")) {
      if (!raw) continue;
      if (raw.endsWith("\r")) raw = raw.slice(0, -1);
      if (raw.startsWith(":") || raw.trim() === "") continue;
      if (!raw.startsWith("data: ")) continue;
      const jsonStr = raw.slice(6).trim();
      if (jsonStr === "[DONE]") continue;
      try {
        const parsed = JSON.parse(jsonStr);
        const content = parsed.choices?.[0]?.delta?.content as string | undefined;
        if (content) onDelta(content);
      } catch { /* ignore partial leftovers */ }
    }
  }

  onDone();
}
```

### React Usage Pattern

```typescript
const [messages, setMessages] = useState<Msg[]>([]);
const [isLoading, setIsLoading] = useState(false);

const send = async (input: string) => {
  const userMsg: Msg = { role: "user", content: input };
  setMessages(prev => [...prev, userMsg]);
  setIsLoading(true);

  let assistantSoFar = "";
  const upsertAssistant = (nextChunk: string) => {
    assistantSoFar += nextChunk;
    setMessages(prev => {
      const last = prev[prev.length - 1];
      if (last?.role === "assistant") {
        // Replace content of last assistant message
        return prev.map((m, i) => (i === prev.length - 1 ? { ...m, content: assistantSoFar } : m));
      }
      // First assistant token: create assistant message
      return [...prev, { role: "assistant", content: assistantSoFar }];
    });
  };

  try {
    await streamChat({
      messages: [...messages, userMsg],
      onDelta: (chunk) => upsertAssistant(chunk),
      onDone: () => setIsLoading(false),
    });
  } catch (e) {
    console.error(e);
    setIsLoading(false);
  }
};
```

### Common Pitfalls to Avoid

1. ❌ Don't buffer full SSE events by splitting on `"\n\n"`
2. ❌ Don't drop lines starting with `":"`
3. ❌ Don't assume every SSE line contains valid JSON
4. ❌ Don't push new assistant message on every token
5. ❌ Don't expose secrets or call external models from client
6. ❌ Always handle `"[DONE]"`, CRLF, final buffer flush, network errors (429/402)
7. ❌ Update `config.toml` to include function and set `verify_jwt` correctly
